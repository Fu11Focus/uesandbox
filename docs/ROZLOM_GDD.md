# РОЗЛОМ — Game Design Document
**Жанр:** PvPvE survival arena (Battle Royale-lite)
**Рушій:** Unreal Engine 5
**Платформа:** PC (Windows / Mac)
**Ціль MVP:** грабельно з 2–6 друзями через Listen Server

---

## Концепція

72 роки тому тектонічна катастрофа «Великий Розлом» розколола земну кору по всій планеті. З тріщин вийшли **Колосси** — масивні біомеханічні істоти, що живляться електромагнітним полем. Міста впали. Людство відступило в укріплені анклави.

У руїнах між зонами залишились ресурси, зброя і таємниці старого світу. Туди відправляють тих, кого називають **Розломниками**.

**Tagline:** Людство вижило. Земля — ні.

---

## Філософія scope

Це проєкт команди з 2 людей. Всі дизайнерські рішення підпорядковані одному правилу: **гра має стати грабельною з друзями якнайшвидше**, а не через 20 місяців. Замість великої battle-royale-копії ми будуємо **сесійну PvPvE арену на 4–8 гравців**, в яку можна грати в сквадах по 2–3 через простий host/join.

Усе, що заважає першій грабельній версії, відкладається або викидається.

---

## Ключові механіки (MVP)

### Матчева петля
1. Гравці з'являються в руїнах розкиданими точками
2. Збирають зброю та фрагменти Панциря з контейнерів
3. Після таймера прокидається Колосс і починає полювати
4. Сейсмічна зона поступово стискає карту, шкодячи поза межами
5. Перемагає останній живий гравець / загін

### Сейсмічна зона
Проста зона-коло, що стискається за таймером. Поза межами — DoT-шкода.

### Колосс
Один AI-бос, що ходить картою, реагує на гравців, атакує. Для MVP достатньо одного типу — **Свердла** або **Плющі** (простіше анімаційно).

### Панцир (MVP: 1–2 класи)
Екзокостюм з однією активною здібністю на cooldown'і. Збирається з компонентів у світі.

## Карта

- **MVP:** 500×500 м, одна локація — зруйноване місто
- **v1.x:** додати 1–2 POI (термальна зона, ліс)
- **v2.0:** 1×1 км з кількома біомами

Nanite + Lumen від старту — для 500×500м це працює навіть на слабких машинах.

## План розробки

### Фаза 0 — Фундамент (поточний стан) ✅
- Третьосторонній темплейт
- Blueprint AIController + BT + Blackboard для Колосса
- Колосс ганяється за найближчим гравцем
- Melee-удар при overlap

### Фаза 1 — Перший грабельний білд (4–8 тижнів)
**Ціль:** запустити Listen Server і грати вдвох.

1. **Health system** — HP у гравця і Колосса, UI-бар, смерть, респавн як спостерігач
2. **Базова зброя** — одна рушниця з hit-scan стрільбою і репликацією damage
3. **Loot pickup** — контейнери зі зброєю/аптечками, інвентар-слот
4. **Сейсмічна зона** — круг що стискається, DoT поза межами
5. **Multiplayer-ready** — усі механіки працюють через RepNotify / Server RPC
6. **Host/Join через Session** — гравець тисне "Host", інший — "Join by IP"

**Критерій завершення фази:** двоє гравців можуть зайти в одну сесію, пограти раунд, один перемагає, рестарт.

### Фаза 2 — Панцир і друге м'ясо (4–6 тижнів)
7. **Кореневий Панцир** — перший клас з активною здібністю (Q — пастка-коріння)
8. **Компоненти у світі** — збираєш N фрагментів → крафтиш Панцир
9. **Стеля шкоди Колосса** — можна нарешті вбити, випадає Ядро
10. **Друга зброя** — пістолет або снайперка, вибір під клас
11. **Basic loadout UI** — екран між матчами

**Критерій:** 4 гравці в одній сесії, є сенс кооперуватись проти Колосса і зраджувати після його смерті.

### Фаза 3 — Карта і друга істота (6–8 тижнів)
12. **Карта 500×500м** — руїни, Nanite-меші, прохідні будинки
13. **NavMesh-генерація** з runtime-оновленням (для Свердла)
14. **Свердло** — другий Колосс, підземна механіка
15. **Буровий Панцир** — другий клас
16. **Audio pass** — MetaSound для кроків, зброї, Колосса

**Критерій:** один повноцінний матч на 4–6 гравців з вибором Панциря і двома можливими Колоссами.

### Фаза 4 — Поліш до "грати з друзями" (4 тижнів)
17. **Game feel** — camera shake, hit markers, damage numbers, VFX попадань
18. **Main menu** — Host / Join / Settings
19. **Налаштування** — чутливість, гучність, роздільна здатність
20. **Playtest з друзями** — збираєш 4–6 друзів, граєш, збираєш фідбек
21. **Balance pass** — урон, швидкість Колосса, таймери зони

**Критерій:** можна запросити друзів у Discord, за 2 хвилини всі в грі, грають годину не нудьгуючи.

### Фаза 5+ — Далі залежно від фідбеку
- Більше Колоссів / Панцирів за пріоритетом з дизайн-пула
- Dedicated Server якщо набереться стабільна група 6+ гравців
- Steam реліз (без EOS, просто Steam Networking / Sessions)
- Anti-cheat, matchmaking, інфраструктура — тільки якщо є живі гравці що цього потребують

## Що свідомо викинуто з scope

Ці речі гарні, але вбивають проєкт для команди з 2 людей:

- ❌ **60 гравців** — Listen Server тримає 4–8 комфортно, 16 з натяжкою
- ❌ **Dedicated Linux server + CI/CD** — не потрібно поки немає хоча б 10 постійних гравців
- ❌ **EOS matchmaking** — Steam invite або IP працює для друзів
- ❌ **EAC** — не має сенсу без публічного релізу
- ❌ **Механіка Злиття (стати Колоссом)** — технічно найскладніша фіча в документі (runtime scale change, нова camera rig, нові ability set'и). Повертається у v2.0 коли буде ресурс.
- ❌ **Всі 6 Колоссів одразу** — MVP це 1, потім +1 кожні 2–3 місяці
- ❌ **Destruction на Chaos** — складно оптимізувати, VFX-ілюзія руйнування простіша
- ❌ **Карта 3×3 км** — поки що не потрібно, 500м вистачає на 30-хвилинний матч 4–6 гравцям

## Ризики і як їх закрити

1. **Netcode для Колосса** — один великий актор реплікується легко на Listen Server. Ризик реальний тільки на 20+ гравцях, тобто поза MVP.
2. **Вигорання** — найбільший ризик для 2-людської команди на довгому проєкті. Вирішення: кожна фаза закінчується грабельним білдом, щоб була віддача.
3. **Scope creep** — свербить додати "ще одну фічу". Правило: додавання фічі у план тільки після плейтесту, а не з голови.
4. **Mac development** — довга компіляція C++, відсутність DX12. Вирішення: триматись Blueprint де можливо, C++ тільки для гарячих місць. Dedicated Server (якщо до нього дійдемо) — Linux через cloud runner, не локально.

---

## Принципи, які не обговорюються

1. **Кожна фаза закінчується грабельним білдом.** Не "в теорії грабельним", а "ми з другом щойно зіграли раунд".
2. **Якщо не грабельно з 2–4 друзями — не додаємо фічі, а фіксимо що зламано.**
3. **Blueprint поки що default.** C++ — тільки коли Blueprint упирається в перформанс або API.
4. **Жодної інфраструктури до того як геймплей click'ає.**
5. **Дизайн-пул живе у GDD як ідеї, а не як обіцянки.**
